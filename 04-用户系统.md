# 第四章：用户系统

> 几乎所有应用都需要用户系统，这一章讲清楚怎么设计才安全。

---

## 4.1 用户体系：注册 / 登录

### 基本流程

```
注册流程：
 1. 用户输入用户名、密码
 2. 验证用户名不重复
 3. 密码加密存储
 4. 返回成功，让用户登录

登录流程：
 1. 用户输入用户名、密码
 2. 根据用户名查数据库
 3. 对比密码哈希
 4. 对了 → 发 Token
 5. 错了 → 返回错误
```

### 代码示例（Python / FastAPI）

**❌ 错误做法：明文存密码**
```python
# 千万不要这么做！！！
@app.post("/register")
def register(username: str, password: str):
    # 明文存密码，数据库泄露所有人密码都没了
    db.execute("INSERT INTO users (username, password) VALUES (?, ?)",
              (username, password))
```

**✅ 正确做法：密码加盐哈希**

```python
# 用 bcrypt 加密，自动加盐
import bcrypt

@app.post("/register")
def register(username: str, password: str):
    # 1. 检查用户名是否已存在
    existing = db.query_one("SELECT id FROM users WHERE username = ?",
                           (username,))
    if existing:
        raise ValueError("用户名已存在")

    # 2. 密码加密
    password_bytes = password.encode('utf-8')
    salt = bcrypt.gensalt()
    password_hash = bcrypt.hashpw(password_bytes, salt)

    # 3. 存数据库（只存哈希，不存明文）
    db.execute(
        "INSERT INTO users (username, password_hash) VALUES (?, ?)",
        (username, password_hash)
    )
    return {"status": "ok"}
```

**登录验证**：
```python
@app.post("/login")
def login(username: str, password: str):
    # 1. 查用户
    user = db.query_one(
        "SELECT id, password_hash FROM users WHERE username = ?",
        (username,)
    )
    if not user:
        raise ValueError("用户名或密码错误")

    # 2. 验证密码
    password_bytes = password.encode('utf-8')
    if not bcrypt.checkpw(password_bytes, user['password_hash'].encode()):
        raise ValueError("用户名或密码错误")

    # 3. 对了，发 Token
    token = create_jwt_token(user_id=user['id'])
    return {"token": token}
```

### 注册验证要点

| 验证项 | 规则 |
|--------|------|
| 用户名 | 3-20 位，只能字母数字下划线 |
| 密码 | 最少 8 位，不允许太简单（比如 123456） |
| 邮箱 | 验证格式 |
| 用户名/邮箱唯一 | 不能重复注册 |

## 4.2 权限控制：谁能做什么

### 基本概念

- **认证（Authentication）**：你是谁？ → 登录
- **授权（Authorization）**：你能做什么？ → 权限

### 常见权限模型

#### 1. 简单系统：RBAC（基于角色的权限控制）

**最常用，推荐新手从这个开始**

```
用户 → 角色 → 权限
```

示例：

| 角色 | 权限 |
|------|------|
| 普通用户 | 只能看自己的待办，只能改自己的 |
| 管理员 | 能看所有人的，能删任何用户 |

代码示例：

```python
# 判断权限：只能修改自己的待办
@app.put("/todos/{id}")
def update_todo(
    id: int,
    content: str,
    current_user = Depends(get_current_user)
):
    # 查这个待办是谁的
    todo = db.query_one("SELECT user_id FROM todos WHERE id = ?", (id,))

    # 权限判断
    if todo.user_id != current_user.id and not current_user.is_admin:
        # 不是我的，也不是管理员 → 拒绝
        raise Forbidden("你没有权限修改这个待办")

    # 有权限，继续修改
    ...
```

#### 2. 更复杂：细粒度权限

比如：文档共享，谁能看，谁能编辑，谁能删除。

那就在中间表存：

```
user_permissions:
- user_id
- document_id
- permission: read/write/delete
```

### 权限设计最佳实践

1. **先简单，后复杂**
   - 刚开始只要一个 `is_admin` 布尔字段就够了
   - 不够了再加角色
   - 不够了再加细粒度

2. **尽早判断**
   - 在 API 入口就判断权限
   - 不要等到数据库操作了才判断

3. **默认拒绝**
   - 默认没有权限
   - 明确允许的才能做

## 4.3 安全机制：用户数据加密

### 密码安全要点

1. **绝对不要明文存密码** ✅
   - 只存哈希，不存明文
   - 用 bcrypt / Argon2 这样的慢哈希算法
   - 不要用 MD5 / SHA1，太快了容易被爆破解

2. **bcrypt 已经自动加盐** ✅
   - 不用你自己加盐，它自动处理了
   - 相同密码，每次加密结果不一样

3. **不要自己发明加密算法** ❌
   - 用成熟库，bcrypt 就很好
   - 密码学水很深，新手玩不转

### 敏感数据保护

哪些是敏感数据？
- 密码
- 身份证号
- 手机号
- 银行卡号
- Token / Secret Key

怎么保护？

| 场景 | 做法 |
|------|------|
| 密码 | bcrypt 哈希，不存明文 |
| Token 返回给前端 | HTTPS 传输，不要记日志 |
| 日志 | 打印日志时脱敏 |
| 数据库返回给前端 | 不要把 password_hash 返回给前端 |

脱敏示例：
```python
def log_user(user):
    # 手机号脱敏：138****1234
    phone = user.phone
    masked_phone = phone[:3] + '****' + phone[-4:]
    logger.info(f"user: {user.name}, phone: {masked_phone}")
```

### HTTPS！HTTPS！HTTPS！

重要的事说三遍：
- **生产环境必须用 HTTPS**
- 不用 HTTPS，中间人能窃听 Token 和密码
- 现在申请证书免费，Let's Encrypt

## 4.4 Token 认证：保持登录状态

### 为什么需要 Token？

HTTP 是无状态的，你登录了一次，下一次请求服务器不知道你是谁。

Token 就是给你发一个"门票"，下次请求带门票，服务器就认识你了。

### JWT 最常用

JWT = JSON Web Token

结构：
```
header.payload.signature
```
- header: 算法等信息
- payload: 存数据（比如 user_id, 过期时间）
- signature: 签名，防篡改

### 使用流程

```
 1. 登录成功 → 生成 JWT Token → 返回给前端
 2. 前端存在 localStorage / sessionStorage
 3. 前端每次请求，都在 Header 带上：
    Authorization: Bearer <token>
 4. 后端验证 Token 对不对，拿出 user_id
 5. 知道是谁了，处理请求
```

### 代码示例（Python）

```python
import jwt
from datetime import datetime, timedelta

# 密钥，存在环境变量里，不要硬编码
SECRET_KEY = settings.jwt_secret
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 30 * 24 * 60  # 30 天过期

def create_jwt_token(user_id: int) -> str:
    """生成 JWT Token"""
    expire = datetime.utcnow() + timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES)
    payload = {
        "user_id": user_id,
        "exp": expire,
        "iat": datetime.utcnow()
    }
    encoded_jwt = jwt.encode(payload, SECRET_KEY, algorithm=ALGORITHM)
    return encoded_jwt

def verify_jwt_token(token: str) -> int:
    """验证 Token，返回 user_id"""
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        user_id = payload.get("user_id")
        if user_id is None:
            raise ValueError("Token 无效")
        return user_id
    except jwt.ExpiredSignatureError:
        raise ValueError("Token 已过期，请重新登录")
    except jwt.JWTError:
        raise ValueError("Token 无效")
```

### FastAPI 依赖获取当前用户

```python
from fastapi import Depends, HTTPException, status
from fastapi.security import OAuth2PasswordBearer

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="login")

def get_current_user(token: str = Depends(oauth2_scheme)) -> User:
    """依赖：获取当前登录用户"""
    try:
        user_id = verify_jwt_token(token)
        user = db.query_one("SELECT * FROM users WHERE id = ?", (user_id,))
        if user is None:
            raise HTTPException(
                status_code=status.HTTP_401_UNAUTHORIZED,
                detail="用户不存在"
            )
        return User(**user)
    except ValueError as e:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail=str(e)
        )

# 使用
@app.get("/me")
def get_me(current_user: User = Depends(get_current_user)):
    """获取当前用户信息"""
    return {
        "id": current_user.id,
        "username": current_user.username,
        "email": current_user.email
    }
```

### Token 存储选择

| 存储位置 | 优点 | 缺点 | 推荐 |
|----------|------|------|------|
| localStorage | 持久化，刷新不丢 | XSS 容易偷 | ✅ 一般项目可用 |
| sessionStorage | 关闭就清，安全 | 刷新就没，麻烦 | ❌ 不推荐 |
| HttpOnly Cookie | XSS 拿不到，安全 | CSRF 需要防 | ✅ 更安全 |

新手推荐：先用 localStorage 简单，做对了再考虑 Cookie。

### Token 过期

- 一定要设过期时间
- 不想让用户经常登录 → 设长一点，30天没问题
- 敏感操作（改密码、支付） → 要求重新登录

### Token 黑名单（退出登录）

JWT 是无状态的，默认不能注销。要实现退出登录：

方案 1（简单）：前端清掉 Token 就完了
- 优点：简单
- 缺点：Token 没过期还能用，如果泄露了还能用

方案 2（安全）：维护一个 Token 黑名单，存在 Redis
```python
def logout(token: str):
    # 把 token 加入黑名单，过期之前都不认
    redis.set(f"blacklist:{token}", "1", ex=token_expire)

def verify(token: str):
    if redis.exists(f"blacklist:{token}"):
        return invalid
```
- 优点：安全，真退出了
- 缺点：需要 Redis，多一点复杂度

## 第四章 总结

### 核心要点

| 要点 | 一句话总结 |
|------|-----------|
| **密码存储** | 绝对不要明文存，用 bcrypt 哈希 |
| **认证授权** | 认证 → 你是谁，授权 → 你能做什么 |
| **权限控制** | 简单项目先加 is_admin，不够了再 RBAC |
| **数据加密** | 敏感数据要脱敏，日志不能打明文 |
| **Token 认证** | JWT 最常用，每次请求带 Header |
| **HTTPS** | 生产必须用，防止窃听 |

### 避开 vibecoding 陷阱

❌ 明文存密码，数据库泄露全完了
❌ 用 MD5 存密码，容易被破解
❌ 权限判断不做，用户能改别人的数据
❌ Token 不设过期，泄露了一直能用
❌ 生产不用 HTTPS，中间人偷密码
❌ 把 password_hash 返回给前端

✅ bcrypt 哈希存密码
✅ 尽早权限判断，默认拒绝
✅ Token 正确过期，退出正确处理
✅ 敏感数据脱敏，不打日志
✅ 生产 HTTPS，证书免费一定要上
